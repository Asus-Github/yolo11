## 一、Triplet Attention 集成

### 1.1 原理回顾（一句话）
Triplet Attention 用 3 条**并行**分支，分别建模 (C×H)、(C×W)、(H×W) 维度的注意力，最后取均值融合。**参数量极小**（只有 3 个 7×7 卷积），适合插在 Backbone 末段。

### 1.2 代码：追加到 `ultralytics/nn/modules/conv.py` 末尾

```python
class ZPool(nn.Module):
    """Z-Pool: 拼接最大池化和平均池化结果（沿指定维度）."""

    def __init__(self, dim=1):
        super().__init__()
        self.dim = dim

    def forward(self, x: torch.Tensor) -> torch.Tensor:
        return torch.cat(
            (torch.max(x, self.dim, keepdim=True)[0], torch.mean(x, self.dim, keepdim=True)),
            dim=self.dim,
        )


class AttentionGate(nn.Module):
    """Triplet Attention 的单个分支：Z-Pool → 7x7 Conv → BN → Sigmoid."""

    def __init__(self, kernel_size: int = 7):
        super().__init__()
        padding = (kernel_size - 1) // 2
        self.compress = ZPool(dim=1)
        self.conv = nn.Conv2d(2, 1, kernel_size=kernel_size, stride=1, padding=padding, bias=False)
        self.bn = nn.BatchNorm2d(1)
        self.sigmoid = nn.Sigmoid()

    def forward(self, x: torch.Tensor) -> torch.Tensor:
        x_compress = self.compress(x)
        x_out = self.bn(self.conv(x_compress))
        return x * self.sigmoid(x_out)


class TripletAttention(nn.Module):
    """Triplet Attention 三元组注意力模块.

    References:
        Misra et al. "Rotate to Attend: Convolutional Triplet Attention Module." WACV 2021.
        https://github.com/landskape-ai/triplet-attention
    """

    def __init__(self, c1: int, c2: int = None, no_spatial: bool = False, kernel_size: int = 7):
        """
        Args:
            c1 (int): 输入通道数（YAML 解析时由 ch[f] 自动传入）.
            c2 (int): 兼容 parse_model 调用，会被忽略；输出通道 = 输入通道.
            no_spatial (bool): 是否关闭 H×W 空间分支（默认 False，三支都开）.
            kernel_size (int): 内部卷积核大小，默认 7.
        """
        super().__init__()
        self.cw = AttentionGate(kernel_size)  # C×W 分支：旋转 (C, W) 维度
        self.hc = AttentionGate(kernel_size)  # C×H 分支：旋转 (C, H) 维度
        self.no_spatial = no_spatial
        if not no_spatial:
            self.hw = AttentionGate(kernel_size)  # H×W 分支：原始空间分支

    def forward(self, x: torch.Tensor) -> torch.Tensor:
        # 分支 1：C×W（先 permute 把 C 和 W 互换）
        x_perm1 = x.permute(0, 2, 1, 3).contiguous()
        x_out1 = self.cw(x_perm1).permute(0, 2, 1, 3).contiguous()

        # 分支 2：C×H（先 permute 把 C 和 H 互换）
        x_perm2 = x.permute(0, 3, 2, 1).contiguous()
        x_out2 = self.hc(x_perm2).permute(0, 3, 2, 1).contiguous()

        # 分支 3：H×W（原始顺序）
        if not self.no_spatial:
            x_out3 = self.hw(x)
            return (x_out1 + x_out2 + x_out3) / 3.0
        return (x_out1 + x_out2) / 2.0
```

## 二、DyHead 集成

### 2.1 原理回顾
DyHead 把"尺度感知 → 空间感知（可变形卷积） → 任务感知"三种注意力**级联**，作用在多尺度特征图上。我们用一个简化版（去掉可变形卷积依赖，避免编译麻烦），便于小白跑通。**论文里写**时仍然按照标准 DyHead 三级公式描述即可。

> **强烈建议小白用简化版**：标准 DyHead 用到 `DCNv2`（可变形卷积），需要从 mmcv 编译 CUDA 算子，对 Mac/无 GPU 环境极不友好。**简化版用 3×3 卷积代替可变形卷积，效果略差但能跑通**，足够发小论文。

### 2.2 代码：追加到 `ultralytics/nn/modules/block.py` 末尾

打开 `/Users/asus/ultralytics/ultralytics/nn/modules/block.py`，**追加到文件末尾**：

```python
class DyHeadBlock(nn.Module):
    """简化版 Dynamic Head Block：尺度感知 + 空间感知 + 任务感知三级注意力.

    References:
        Dai et al. "Dynamic Head: Unifying Object Detection Heads with Attentions." CVPR 2021.
    Notes:
        本实现去掉了 DCNv2（可变形卷积），用 3x3 标准卷积替代，便于在无 mmcv 环境下使用。
    """

    def __init__(self, in_channels: int, out_channels: int = None):
        super().__init__()
        out_channels = out_channels or in_channels
        self.in_channels = in_channels
        self.out_channels = out_channels

        # 空间感知（去 DCN，用普通 3x3）
        self.spatial_conv = nn.Conv2d(in_channels, out_channels, kernel_size=3, padding=1, bias=True)
        self.norm = nn.GroupNorm(16, out_channels) if out_channels >= 16 else nn.GroupNorm(1, out_channels)

        # 尺度感知注意力：对各层的 1x1 GAP 做 hard-sigmoid 加权
        self.scale_attn = nn.Sequential(
            nn.AdaptiveAvgPool2d(1),
            nn.Conv2d(out_channels, 1, 1),
            nn.ReLU(inplace=True),
            nn.Hardsigmoid(),
        )

        # 任务感知注意力：动态 ReLU 风格（4 个可学习参数 α1, β1, α2, β2）
        self.task_attn = nn.Sequential(
            nn.AdaptiveAvgPool2d(1),
            nn.Conv2d(out_channels, max(out_channels // 4, 1), 1),
            nn.ReLU(inplace=True),
            nn.Conv2d(max(out_channels // 4, 1), out_channels * 4, 1),
        )

    def forward(self, x: torch.Tensor) -> torch.Tensor:
        # ① 空间感知
        feat = self.norm(self.spatial_conv(x))

        # ② 尺度感知（通道-空间联合权重）
        scale_w = self.scale_attn(feat)
        feat = feat * scale_w

        # ③ 任务感知（动态 ReLU）
        task_w = self.task_attn(feat).view(feat.shape[0], -1, 1, 1)
        a1, b1, a2, b2 = task_w.chunk(4, dim=1)
        a1 = a1 * 2.0 - 1.0  # 范围 [-1, 1]
        a2 = a2 * 2.0 - 1.0
        feat = torch.maximum(feat * a1 + b1, feat * a2 + b2)

        return feat
```

```python
class DyHeadDetect(Detect):
    """带 DyHead 模块的检测头：在原 Detect 前级联 N 个 DyHeadBlock.

    Note:
        参数顺序必须与父类 Detect 完全一致 (nc, reg_max, end2end, ch)，
        因为 parse_model 里 args.extend([reg_max, end2end, [ch[x] for x in f]])
        会按这个顺序追加。num_blocks 通过类属性配置，不进 __init__ 形参，
        避免 parse_model 调用错位。
    """

    # 类属性：DyHead 堆叠数；YAML 里若想改，可在 head.py 顶部或 train 脚本里改
    num_blocks: int = 2

    def __init__(self, nc: int = 80, reg_max: int = 16, end2end: bool = False, ch: tuple = ()):
        super().__init__(nc=nc, reg_max=reg_max, end2end=end2end, ch=ch)
        # 为每个 FPN 层级独立准备 num_blocks 个 DyHeadBlock
        self.dyhead = nn.ModuleList(
            nn.Sequential(*[DyHeadBlock(c, c) for _ in range(self.num_blocks)]) for c in ch
        )

    def forward(self, x):
        # 先经过 DyHead，再交给原 Detect
        x = [self.dyhead[i](x[i]) for i in range(self.nl)]
        return super().forward(x)
```

> **为什么不把 num_blocks 放到 `__init__`**：ultralytics 的 `parse_model` 对 Detect 系列检测头会自动 `args.extend([reg_max, end2end, [ch[x] for x in f]])`，把 `(nc, reg_max, end2end, ch)` 按顺序填到形参。如果在中间插入一个 `num_blocks`，会把 `reg_max` 错位塞到 `num_blocks`，导致检测头输出维度全错。**改动 num_blocks 请改类属性**：训练前 `from ultralytics.nn.modules.head import DyHeadDetect; DyHeadDetect.num_blocks = 3`。

### 3.1 修改 `bbox_iou` 函数

打开 `/Users/asus/ultralytics/ultralytics/utils/metrics.py`，找到第 105 行 `def bbox_iou(`：

**步骤 ①**：在函数签名加入 `WIoU` 参数（默认 False）：

```python

    box1: torch.Tensor,
    box2: torch.Tensor,
    xywh: bool = True,
    GIoU: bool = False,
    DIoU: bool = False,
    CIoU: bool = False,
    WIoU: bool = False,                    # ← 新增
    scale: bool = False,                   # ← 新增（控制是否返回离群度 scaled 系数）
    eps: float = 1e-7,
) -> torch.Tensor:
```

**步骤 ②**：在 `return iou  # IoU` 这行（约第 171 行）**之前**，加入 WIoU 分支：

```python
    if WIoU:
        # WIoU v3: 距离注意力 + 动态非单调聚焦
        cw = b1_x2.maximum(b2_x2) - b1_x1.minimum(b2_x1)
        ch_ = b1_y2.maximum(b2_y2) - b1_y1.minimum(b2_y1)
        c2 = cw.pow(2) + ch_.pow(2) + eps
        # 中心点距离平方
        rho2 = (
            (b2_x1 + b2_x2 - b1_x1 - b1_x2).pow(2)
            + (b2_y1 + b2_y2 - b1_y1 - b1_y2).pow(2)
        ) / 4
        # 距离注意力 R_WIoU（梯度从 c2 切断；clamp 防 AMP 下指数溢出 → NaN）
        r_wiou = torch.exp((rho2 / c2.detach()).clamp_(max=10.0))
        if scale:
            return iou, r_wiou  # 调用方再算聚焦因子
        return iou, r_wiou
    return iou  # IoU
```

> **解释**：返回 `(iou, r_wiou)` 元组，让调用方在 BboxLoss 里再做"动态非单调聚焦"，因为这一步需要 batch 内所有样本的 `loss_iou_mean` 才能算出来。
> **AMP 安全说明**：`torch.exp` 在 fp16 下输入 > 11.09 就溢出为 inf，**必须 clamp**。clamp 上界 10 对应 `exp(10)≈22026`，足够大且不会溢出 fp16 范围（最大 65504）。

### 3.2 修改 `BboxLoss` 启用 WIoU

打开 `/Users/asus/ultralytics/ultralytics/utils/loss.py`：

**步骤 ①**：在文件顶部 `import` 区域确认有 `from ultralytics.utils.metrics import bbox_iou`（应该已经有）。

**步骤 ②**：找到 `class BboxLoss`（第 110 行），改写如下：

```python
class BboxLoss(nn.Module):
    """Criterion class for computing training losses for bounding boxes.

    切换 IoU 类型：
        BboxLoss.iou_type = "wiou"   # 启用 Wise-IoU v3
        BboxLoss.iou_type = "ciou"   # baseline (CIoU)
    在 import YOLO 之后、model.train() 之前赋值即可。
    """

    # 类属性：消融切换用
    iou_type = "wiou"      # "ciou" / "wiou"
    wiou_alpha = 1.9
    wiou_delta = 3.0

    def __init__(self, reg_max: int = 16):
        super().__init__()
        self.dfl_loss = DFLoss(reg_max) if reg_max > 1 else None
        # iou_mean 用 buffer 而非类属性：
        #   - 自动跟随模型 .to(device)，无须手动管理设备
        #   - 自动存进 .pt checkpoint，恢复训练时连续
        #   - 多个模型实例互不干扰（类属性会跨实例共享）
        self.register_buffer("iou_mean", torch.tensor(1.0))

    def forward(
        self,
        pred_dist, pred_bboxes, anchor_points, target_bboxes,
        target_scores, target_scores_sum, fg_mask, imgsz, stride,
    ):
        weight = target_scores.sum(-1)[fg_mask].unsqueeze(-1)

        if self.iou_type == "wiou":
            iou, r_wiou = bbox_iou(
                pred_bboxes[fg_mask], target_bboxes[fg_mask],
                xywh=False, WIoU=True,
            )
            loss_iou_per = (1.0 - iou)                                    # [N, 1]

            # 滑动均值：fp32 计算避免 AMP 下 fp16 溢出
            with torch.no_grad():
                cur_mean = loss_iou_per.detach().float().mean().clamp_(min=1e-7)
                self.iou_mean.mul_(0.9).add_(cur_mean * 0.1)

            beta = (loss_iou_per.detach() / self.iou_mean.clamp_(min=1e-7)).clamp_(min=1e-7)
            # 动态非单调聚焦因子 r = β / (δ · α^(β-δ))
            r = beta / (self.wiou_delta * (self.wiou_alpha ** (beta - self.wiou_delta)))
            loss_iou = ((r * r_wiou * loss_iou_per) * weight).sum() / target_scores_sum
        else:  # ciou (baseline)
            iou = bbox_iou(
                pred_bboxes[fg_mask], target_bboxes[fg_mask],
                xywh=False, CIoU=True,
            )
            loss_iou = ((1.0 - iou) * weight).sum() / target_scores_sum

        # DFL loss（保持原样不动）
        if self.dfl_loss:
            target_ltrb = bbox2dist(anchor_points, target_bboxes, self.dfl_loss.reg_max - 1)
            loss_dfl = self.dfl_loss(
                pred_dist[fg_mask].view(-1, self.dfl_loss.reg_max),
                target_ltrb[fg_mask],
            ) * weight
            loss_dfl = loss_dfl.sum() / target_scores_sum
        else:
            target_ltrb = bbox2dist(anchor_points, target_bboxes)
            target_ltrb = target_ltrb * stride
            target_ltrb[..., 0::2] /= imgsz[1]
            target_ltrb[..., 1::2] /= imgsz[0]
            pred_dist = pred_dist * stride
            pred_dist[..., 0::2] /= imgsz[1]
            pred_dist[..., 1::2] /= imgsz[0]
            loss_dfl = (
                F.l1_loss(pred_dist[fg_mask], target_ltrb[fg_mask], reduction="none").mean(-1, keepdim=True) * weight
            )
            loss_dfl = loss_dfl.sum() / target_scores_sum

        return loss_iou, loss_dfl
```

> **关键差异 vs 旧版**：
> - `iou_type` 仍是**类属性**（最简方式，单卡 5090 完全够用）
> - `iou_mean` 是**实例 buffer**（不是类属性 `_iou_mean`）。原因：buffer 会自动 `.to('cuda:0')`、自动存进 checkpoint、不会跨模型实例污染。比类属性 `_iou_mean = 1.0` 干净。
> - WIoU 的 `exp` 已在 `bbox_iou` 里 `clamp_(max=10)`，AMP 下不会 NaN。

```yaml
nc: 80

  - [-1, 1, Conv, [1024, 3, 2]]            # 7  P5/32
  - [-1, 2, C3k2, [1024, True]]            # 8
  - [-1, 1, TripletAttention, []]          # 9  ★ 新增：P5 末端 Triplet
  - [-1, 1, SPPF, [1024, 5]]               # 10
  - [-1, 2, C2PSA, [1024]]                 # 11

head:
  - [-1, 1, nn.Upsample, [None, 2, "nearest"]]   # 12
  - [[-1, 6], 1, Concat, [1]]                    # 13
  - [-1, 2, C3k2, [512, False]]                  # 14

  - [-1, 1, nn.Upsample, [None, 2, "nearest"]]   # 15
  - [[-1, 4], 1, Concat, [1]]                    # 16
  - [-1, 2, C3k2, [256, False]]                  # 17  P3/8-small

  - [-1, 1, Conv, [256, 3, 2]]                   # 18
  - [[-1, 14], 1, Concat, [1]]                   # 19
  - [-1, 2, C3k2, [512, False]]                  # 20  P4/16-medium

  - [-1, 1, Conv, [512, 3, 2]]                   # 21
  - [[-1, 11], 1, Concat, [1]]                   # 22
  - [-1, 2, C3k2, [1024, True]]                  # 23  P5/32-large

  - [[17, 20, 23], 1, DyHeadDetect, [nc]]       # 24  ★ DyHeadDetect (num_blocks 由类属性控制)
```

**关键改动点**：
1. 第 9 层：在 SPPF 前插入 `TripletAttention`（输入输出同通道，不影响后续 index）。
2. 最后一行：`Detect, [nc]` → `DyHeadDetect, [nc]`。**注意只传 `[nc]`，不要写 `[nc, 2]`**——`num_blocks` 已经由类属性 `DyHeadDetect.num_blocks=2` 在 head.py 里定死。如果传 `[nc, 2]` 会被 `parse_model` 解析为 `(nc, reg_max=2, ...)`，导致检测头输出维度全错。
3. 因为第 9 层多了一层，原来的 P3/P4/P5 输出索引 `[16, 19, 22]` → `[17, 20, 23]`（**层数 +1**）。
   > **小白提示**：head 内的 `[-1, X]` 是**相对索引**（往上数 1 层、跳到第 X 层）。`-1` 的部分不需要改，只需要改**绝对索引**：backbone 引用（原 4→5、6→7）和 head 内引用（原 13→14、原 10→11）。本手册 YAML 已经改好，照抄即可。
4. WIoU **不在 YAML 配置**，由环境变量 `IOU_TYPE` 控制。

## 附录 A：Triplet Attention permute 维度说明（小白可跳过）

PyTorch 张量形状 `[B, C, H, W]`：
- `permute(0, 2, 1, 3)` → `[B, H, C, W]`：使原 H 当作"通道"维，原 C 当作"高度"维 → AttentionGate 沿新通道轴（H）做 Z-Pool，等价于 (C, W) 维度的注意力
- `permute(0, 3, 2, 1)` → `[B, W, H, C]`：使原 W 当作"通道"维 → 等价于 (C, H) 维度的注意力
- 默认序：直接做 → 等价于 (H, W) 空间维度的注意力

三支 permute 完做完 AttentionGate，再 permute 回原序求平均，即得 Triplet 输出。

## 附录 B：为何用简化版 DyHead 而非标准版
