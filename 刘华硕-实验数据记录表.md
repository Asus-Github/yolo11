## 一、必须统计的核心指标（按重要程度排序）

### 🔴 一级指标（论文必报，缺一不可）

| 指标 | 中文名 | 单位 | 来源 | 越大越好 |
|------|------|-----|------|---------|
| **mAP@0.5** | 主指标，IoU=0.5 时的平均精度 | % | val 输出 / results.csv `metrics/mAP50(B)` | ✅ |
| **mAP@0.5:0.95** | COCO 主指标（IoU 0.5-0.95 平均） | % | results.csv `metrics/mAP50-95(B)` | ✅ |
| **Precision (P)** | 精确率 | % | results.csv `metrics/precision(B)` | ✅ |
| **Recall (R)** | 召回率 | % | results.csv `metrics/recall(B)` | ✅ |
| **Params** | 模型参数量 | M | model.info() / 终端打印 | ❌ 越小越好 |
| **GFLOPs** | 浮点运算量 | G | model.info() / 终端打印 | ❌ 越小越好 |
| **FPS** | 推理速度 | f/s | val 推理时间 1/preprocess+inference+postprocess | ✅ |

### 🟡 二级指标（论文加分）

| 指标 | 来源 | 用处 |
|------|------|------|
| **F1-score** | (2·P·R)/(P+R)，自己算 | 综合指标 |
| **AP_small** | COCO 小目标专项 | 论文写"小目标改善" |
| **AP_medium / AP_large** | COCO 分尺度 | 论证多尺度有效 |
| **per-class AP** | val 时每类 AP | 类别级精细分析 |
| **训练总时长** | results.csv 时间戳差 | 估算实验成本 |
| **收敛 epoch** | val_loss 最小值的 epoch | 比较收敛速度 |
| **GPU 显存峰值** | nvidia-smi 训练时记录 | 写硬件配置时用 |

### 🟢 三级指标（可选）

| 指标 | 来源 | 用处 |
|------|------|------|
| Inference latency (ms) | 1000 / FPS | 边缘部署论文用 |
| Model size (MB) | 文件大小 `ls -lh best.pt` | 部署友好度 |
| Train_box_loss / Train_cls_loss / Train_dfl_loss | results.csv | 论文绘 loss 曲线 |
| Val_box_loss / Val_cls_loss / Val_dfl_loss | results.csv | 验证集 loss 曲线 |

### 7.3 测 FPS（必须独立测，不能信训练时显示的）

```python
from ultralytics import YOLO
import time, torch
model = YOLO('/root/autodl-tmp/runs/ablation/tdw/weights/best.pt')
model.fuse()  # 融合 conv+bn 加速
img = torch.zeros(1, 3, 640, 640).cuda()

# 预热 50 次（防止冷启动偏差）
for _ in range(50):
    _ = model.predict(img, verbose=False)

torch.cuda.synchronize()
t0 = time.time()
for _ in range(500):
    _ = model.predict(img, verbose=False)
torch.cuda.synchronize()
fps = 500 / (time.time() - t0)
print(f"FPS: {fps:.1f}")
```

### 7.4 一键提取全部消融结果到 CSV

新建 `/root/autodl-tmp/ultralytics/extract_results.py`：

```python
"""遍历 runs/ablation 下所有组，提取关键指标汇总到一个 CSV."""
import pandas as pd
from pathlib import Path

results_root = Path('/root/autodl-tmp/runs/ablation')
rows = []
for exp_dir in sorted(results_root.iterdir()):
    csv_path = exp_dir / 'results.csv'
    if not csv_path.exists():
        continue
    df = pd.read_csv(csv_path)
    last = df.iloc[-1]
    rows.append({
        'name': exp_dir.name,
        'epochs': len(df),
        'P': float(last.get('metrics/precision(B)', 0)),
        'R': float(last.get('metrics/recall(B)', 0)),
        'mAP50': float(last.get('metrics/mAP50(B)', 0)),
        'mAP50_95': float(last.get('metrics/mAP50-95(B)', 0)),
        'box_loss': float(last.get('train/box_loss', 0)),
    })

summary = pd.DataFrame(rows)
summary['F1'] = 2 * summary['P'] * summary['R'] / (summary['P'] + summary['R'] + 1e-7)
summary = summary.round(4)
summary.to_csv('/root/autodl-tmp/runs/ablation/_summary.csv', index=False)
print(summary.to_string(index=False))
```

跑完所有消融后执行：
```bash
python extract_results.py
# 然后 rsync 把 _summary.csv 拉回本地
```

## 九、写论文时的"指标取值约定"

为了避免被审稿人挑刺，统一遵循：

1. **mAP 取 best 而非 last**：用 `weights/best.pt` 在 val 上重新跑一遍 `model.val()` 的输出，**不要直接复制 results.csv 最后一行**（可能不是 best）
2. **FPS 必须 fuse 后测**：`model.fuse()` 后再测，否则会被审稿人指出"未融合 BN"
3. **Params 报融合后**：`model.info()` 默认报融合后的，没问题
4. **batch 必须统一**：所有横向对比都用同一 batch（推荐 32）
5. **seed 必须固定**：`seed=42`，避免随机性导致结果不稳
6. **报告平均值**：如果时间允许，**每组跑 3 个 seed 取均值±std**，论文表更专业。但作为小论文跑一次也可以接受。
